# Как запустить проект Nocode CRUD Platform

Проект состоит из **бэкенда** (Java, Spring Boot) и **фронтенда** (React, Vite, Tailwind CSS). Обе части нужно запустить для полноценной работы приложения вместе с базой данных и reverse-proxy маршрутизатором (Traefik).

У вас есть два варианта запуска:

---

## Вариант 1. Полноценный запуск через Docker (Рекомендуется)

Этот вариант собирает и запускает всю платформу целиком в изолированных контейнерах, точно так же, как это происходит на реальном сервере.

**Требования:** Установленный Docker Desktop запуска.

**Windows (PowerShell):**
В корневой папке проекта есть специальный скрипт запуска `start-platform.ps1`.
Откройте терминал PowerShell в корне проекта и выполните:
```powershell
powershell -ExecutionPolicy Bypass -File .\start-platform.ps1
```
*(Скрипт остановит предыдущие сессии, пересобирает образы и поднимет всю инфраструктуру. Платформа будет доступна без указания портов по основному адресу: `http://localhost`)*

---

## Вариант 2. Запуск для локальной разработки (Dev-режим)

Используйте этот вариант, если вы намерены вносить изменения в исходный код платформы. Этот скрипт запустит только базу данных и Traefik в Docker, а Frontend (Vite) и Backend (Spring Boot) откроются в отдельных окнах терминала для удобной отладки.

**Требования:** Java 21, Node.js 18+, Docker.

**Windows (PowerShell):**
Откройте терминал PowerShell в корне проекта и выполните:
```powershell
powershell -ExecutionPolicy Bypass -File .\start-dev.ps1
```

При этом варианте:
*   Frontend будет доступен по адресу: `http://localhost:5173`
*   Backend API будет доступен по адресу: `http://localhost:8082`
*   Traefik маршруты (для сгенерированных внутри платформы проектов) по-прежнему будут работать.

---
