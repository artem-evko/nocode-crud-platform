## 1. Информация о проекте
**Тема:** NO-CODE ПЛАТФОРМА ДЛЯ АВТОМАТИЗИРОВАННОЙ ГЕНЕРАЦИИ CRUD-ПРИЛОЖЕНИЙ
**Суть:** Разрабатывается платформа, в которой пользователь с помощью визуального графового редактора (узлы — сущности, связи — отношения метамодели) описывает структуру предметной области. Из этой метамодели платформа автоматически генерирует:
1. Конфигурацию и миграции базы данных (через Liquibase).
2. Backend (через Spring Boot, Spring Data REST и кастомные контроллеры MVC, с валидацией Jakarta Validation).
3. Frontend (формы, списки, таблицы) для управления данными (CRUD).

### Выбранный технологический стек (Описан в Главе 2):
*   **Backend:** Java, Spring Boot, Hibernate ORM, PostgreSQL (для данных платформы и сгенерированных приложений, используется JSON/JSONB для хранения метамодели), Liquibase (версионирование БД алгоритмами миграций). API описывается через OpenAPI (Swagger).
*   **Frontend:** React, TypeScript, Vite, React Flow (для визуального редактора), Tailwind CSS (стилизация), Zustand (управление глобальным состоянием).

### Краткое содержание уже написанных глав:
*   **Глава 1:** Обоснован модельно-ориентированный подход, при котором метамодель ("сущность-атрибут-связь") выступает источником истины. Исследованы риски "вендор-лока", вопросы безопасности и производительности.
*   **Глава 2:** Обоснован выбор стека. PostgreSQL + Liquibase обеспечивают прозрачность данных в миграции. Spring Boot даёт промышленную готовность сервера, React Flow — функциональный визуальный конструктор UI без привязки к серверу.
